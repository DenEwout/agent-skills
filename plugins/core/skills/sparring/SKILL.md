---
name: sparring
description: Stress-test an idea, plan, or decision by interrogating it with tough, adversarial questions.
argument-hint: [topic or decision to spar]
---

As a helpful colleague, spar with me on an idea or topic. Interview me using the `AskUserQuestion` tool to understand the topic and challenge me on it with tough, adversarial questions and offer alternative options.

If a question can be answered by exploring the codebase, explore the codebase instead.

Break the topic down into atomic decisions, and resolve dependencies. Ask multiple questions at once when possible. 

When a shared understanding is  reached, ask the user if an output should be using `/htmldoc`.
